---
title: Quickstart — Run SCA Academia from Scratch
description: Step-by-step guide to clone the repo, configure environment variables, seed the database, and launch both the backend and frontend servers.
---

This guide takes you from an empty machine to a fully running SCA Academia instance. You will set up the Node.js/Express backend connected to MongoDB, seed the initial owner and partner accounts, then start the React/Vite frontend. The whole process takes under ten minutes with Node.js and MongoDB already installed.

<Warning>
  The seed script creates three accounts with the default password `admin123`. **Change all passwords before exposing the system to a network or going to production.** See the production checklist in the [architecture guide](/architecture) for the full hardening steps.
</Warning>

<Steps>

  <Step title="Clone the repository">
    Clone the SCA Academia monorepo to your local machine.

    ```bash
    git clone https://github.com/muz4miL/sca-academia.git
    cd sca-academia
    ```
  </Step>

  <Step title="Install backend dependencies">
    The backend lives in the `backend/` directory and uses Express, Mongoose, and several security packages.

    ```bash
    cd backend
    npm install
    ```

    Key packages installed: `express`, `mongoose`, `jsonwebtoken`, `bcryptjs`, `cookie-parser`, `cors`, `dotenv`, `multer`, `nodemon`.
  </Step>

  <Step title="Configure backend environment variables">
    Create a `.env` file inside `backend/` using the template below. All three variables are required for the server to start.

    ```env backend/.env
    # MongoDB connection string — local or Atlas
    MONGODB_URI=mongodb://localhost:27017/academyDB

    # Port the Express server listens on
    PORT=5000

    # JWT signing secret — use a random string of at least 32 characters
    JWT_SECRET=your-super-secret-jwt-key-change-this-in-production

    # Optional: control token lifespan (default 7 days)
    JWT_EXPIRES_IN=7d
    ```

    For **MongoDB Atlas** replace `MONGODB_URI` with your cluster connection string:

    ```env
    MONGODB_URI=mongodb+srv://<user>:<password>@cluster.mongodb.net/academyDB
    ```
  </Step>

  <Step title="Seed the database">
    The seed script creates the three protected core accounts and any required configuration documents.

    ```bash
    npm run seed
    ```

    A successful run prints:

    ```
    ✅ MongoDB Connected Successfully!
    👥 Creating Core Partner Accounts...
    ✅ 3 core users created!
       - Sir Waqar Baig (OWNER)
       - Dr. Zahid (PARTNER)
       - Sir Shah Saud (PARTNER)
    ```

    Default credentials created by the seed:

    | Full Name | Username | Password | Role |
    |---|---|---|---|
    | Sir Waqar Baig | `waqar` | `admin123` | OWNER |
    | Dr. Zahid | `zahid` | `admin123` | PARTNER |
    | Sir Shah Saud | `saud` | `admin123` | PARTNER |
  </Step>

  <Step title="Start the backend server">
    Run the development server with auto-reload via nodemon.

    <CodeGroup>
      ```bash Development (nodemon)
      npm run dev
      ```

      ```bash Production (node)
      npm start
      ```
    </CodeGroup>

    You should see:

    ```
    🚀 Server is running on port 5000
    📡 API available at http://localhost:5000
    ✅ MongoDB Connected Successfully!
    ```

    Verify by visiting `http://localhost:5000` — it returns a JSON manifest of all registered route prefixes.
  </Step>

  <Step title="Install frontend dependencies">
    Open a second terminal window, navigate to `frontend/`, and install the React/Vite application dependencies.

    ```bash
    cd frontend
    npm install
    ```
  </Step>

  <Step title="Configure frontend environment variables">
    Create a `.env` file inside `frontend/` pointing at the running backend.

    ```env frontend/.env
    # Full URL to the backend API — no trailing slash
    VITE_API_URL=http://localhost:5000
    ```

    If you changed `PORT` in the backend `.env`, update this value to match.
  </Step>

  <Step title="Start the frontend dev server">
    ```bash
    npm run dev
    ```

    Vite starts on port **5173** by default:

    ```
    VITE v5.x.x  ready in ~300ms

    ➜  Local:   http://localhost:5173/
    ➜  Network: use --host to expose
    ```

    Open `http://localhost:5173` in your browser.
  </Step>

  <Step title="Log in with the OWNER account">
    On the login page enter the seeded OWNER credentials:

    - **Username:** `waqar`
    - **Password:** `admin123`

    After a successful login you land on the **Dashboard**, which shows live stats: total active students, today's fee collections, pending balances, teacher count, and a recent transaction feed. The sidebar exposes all modules available to the OWNER role — Students, Finance, Teachers, Payroll, Expenses, Timetable, Attendance, Gatekeeper, Inventory, Leads, and more.

    <Tip>
      Navigate to **Users** in the sidebar to change the default passwords for all three seeded accounts before sharing access with partners or staff.
    </Tip>
  </Step>

</Steps>

<Tip>
  To run both servers simultaneously without managing two terminals, use the included PM2 production script from the repo root:

  ```bash
  bash run-server-production.sh
  ```

  This installs PM2 globally if needed, starts `sca-backend` and `sca-frontend` as named PM2 processes, and persists them across reboots. Use `pm2 logs` to tail output and `pm2 stop sca-backend sca-frontend` to shut both down.
</Tip>
